3-Tier AWS Architecture Case Study

3-Tier Secure Cloud Architecture

Designed and deployed a highly available, fault-tolerant 3-tier web architecture on AWS mimicking enterprise deployment standards.

Tech Stack

  • Cloud: AWS EC2, S3, RDS
  • Networking: VPC, Route 53, ALB
  • Scaling: Auto Scaling Groups
  • Security: IAM, Security Groups

Key Features

  • Auto-scaling web servers based on CPU utilization
  • Multi-AZ database deployment
  • Load balancing dynamic traffic distribution
  • Isolated private and public subnets

System Architecture

Internet traffic flows through Route 53 to an Application Load Balancer. ALB distributes requests to EC2 instances in private subnets within a custom VPC. EC2 connects to a Multi-AZ RDS MySQL instance for structured data storage.

Impact / Metrics

  • Achieved theoretical 99.9% uptime capability
  • Zero downtime during simulated failure testing

Challenges & Solutions

Challenge: Ensuring database failover without manual intervention during server crashes.
Solution: Configured RDS Multi-AZ deployment to automatically switch to a standby replica during simulated primary instance failure.
